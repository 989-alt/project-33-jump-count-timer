# 단계 2 · Brainstormer 결과

> 무인 세션 — AI 자체 판단으로 MUST / SHOULD / MUST NOT 결정.

## 사용자 시나리오 (체육 시간 1분)
1. 교사가 TV에 띄움 → 학생 이름을 한 번만 입력 (또는 미리 등록된 명단에서 선택).
2. 시간 프리셋(예: 30초) 선택.
3. "시작" 누름 → 3·2·1 카운트다운 → 점프하며 스페이스바(또는 큰 화면 탭) 연타.
4. 타임업 비프음 → 결과(개수) 노출, 본인 PR 갱신 여부 표시.
5. 다음 학생으로 넘김 → 반복.

## MUST (꼭 필요한 기능, 4개)
1. **시간 프리셋 + 카운트다운**
   - 프리셋 15·30·60·180초 + 임의 입력(5~600초).
   - "시작" 후 3·2·1 → 본 타이머. 종료 시 비프음.
2. **스페이스바 + 큰 화면 탭으로 카운트**
   - 스페이스바: 데스크탑/노트북 + 키보드 연결된 TV.
   - 화면 어디든 큰 영역 탭(터치/마우스): 태블릿/스마트보드 환경.
   - 연타 안티-락(50ms 디바운스만, 너무 짧게 잡으면 빠른 학생 누락).
3. **학생 PR 보드 (localStorage)**
   - 학생 이름 추가/삭제/리네임.
   - 학생별 최고 기록(개수, 시간 프리셋별 분리, 측정 일시).
   - 현재 측정값이 PR 갱신이면 "🏆 신기록!" 시각 알림.
4. **풀스크린 라이브 카운터 + 결과 화면**
   - 측정 중에는 카운트 숫자가 화면 거의 전체(최소 30vw 폰트). TV 멀리서도 보이게.
   - 종료 즉시 결과 패널: 학생명·시간·개수·PR 비교(이전 PR / 갱신 표시).

## SHOULD (있으면 좋은 기능, 3개)
1. **반 PR 보드 export/import (JSON)**
   - 학기 내내 데이터 유지, 새 PC/디바이스로 옮기기. 다운로드/파일 업로드.
2. **반 랭킹 (시간 프리셋별 Top 5)**
   - 각 프리셋에서 누가 1~5등인지 큰 화면에. (이름·기록).
3. **음소거 토글 + 카운트다운 비프음 옵션 (3·2·1)**
   - 도서관 인근 등 음소거 필요 시. 시작 카운트다운 3·2·1 비프, 종료 더블 비프.

## MUST NOT (절대 금지, 3개)
1. **카메라/마이크 접근** — 토픽 명시. 자동 카운트도, 음성 트리거도 X.
2. **외부 네트워크 전송** — 모든 데이터 로컬. 분석/텔레메트리/CDN 호출 0건.
3. **로그인·계정·서버 연동** — 학교 계정 입력 요청 절대 금지, 익명 로컬만.

## scope creep 방지를 위한 추가 제약
- 애니메이션·시각효과 미니멀 유지 (Nike 디자인 톤). 폭죽·반짝이·일러스트 사용 X.
- 차트/그래프 없음. PR은 숫자 + 텍스트로만 표현.
- 다국어 X — 한국어 단일.
- 다크 모드 X — Nike는 흰 캔버스 우선.
